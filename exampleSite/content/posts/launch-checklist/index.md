Before shipping a Hugo theme, it helps to keep the release scope small and
repeatable.

## Minimal release items

1. Add metadata in `theme.toml`.
2. Include a real `exampleSite`.
3. Provide a license and screenshots.
4. Make personal links configurable.

## Next step

After the theme itself is ready, the project can grow a dedicated introduction
page inside Pocket Hugo with cross-links back to the theme repository.
